Act 5 — Engine, Roadmap, Payoff (25:00 – 30:00)
Slide 26 ~25:30
CHANGED v7: Architecture moved here to match PPT position. Updated with PPT agent terminology. Added MCP, caching, OpenSearch.
GOBIND

"You've seen what it does. Now let me show you how it's built. This slide is for the engineers and architects in the room — this is the Technical Architecture, a governed, scalable reasoning layer. I'll walk through each layer top-down."

Experience Layer — React front-end with ADFS SSO for enterprise authentication. Chat UI, file uploads, project dashboards, response rendering. No training required.

Semantic Router & Context Window — this is the first thing that fires. Before the LLM is even invoked, the system analyses intent and dynamically assembles context from the Delivery Graph — pulling code, Jira tickets, architecture docs, prior decisions. The right context, assembled before reasoning begins.

Control Layer — Multi-Agent Orchestration. This is the brain. Agents for every persona — PO Agent, Architect Agent, Reviewer Agent — with workflows, iteration limits, and human-in-the-loop gates. The orchestration runs on LangGraph, deployed as AWS Lambda functions.

Intelligence Layer — three specialised agents working together. The Planner Agent breaks complex queries into sub-tasks. The Execution Agent calls enterprise-grade foundation models via Amazon Bedrock — Claude, GPT-4, model-agnostic by design. The Critic Agent validates every output against known constraints before it goes further.

Knowledge & Action Layer — Amazon Neptune as both graph store and vector memory. Nodes and edges for structural queries. Vector embeddings for semantic search across Confluence, Jira, Jules, JET, ServiceNow, SharePoint. We also use Amazon OpenSearch for full-text search and log analytics — complementing Neptune's graph traversal with keyword-level precision. And MCP — Model Context Protocol — the standardised interface that lets our agents talk to external tools and data sources without custom integrations. MCP means we plug into any enterprise system with a consistent protocol, not bespoke connectors.

Caching Layer — semantic caching at the retrieval and generation layers. If a similar query has been answered recently, the system serves from cache — reducing latency and foundation model costs. Neptune's vector similarity drives cache hits, so you get fast responses without sacrificing freshness.

Enterprise Guardrails — this is the governance lock. Pre-Guard: permission enforcement before any agent acts. Post-Guard: syntax validation, compliance checks, and the Human-in-the-Loop approval gate. Nothing reaches production without human sign-off. And there's a Feedback Loop: every accepted action feeds back into vector memory, so the system gets smarter with use.

Platform Layer — S3 for document storage and artefact persistence. Lambda for serverless compute — no idle servers, scales to zero, scales to thousands concurrently. Full audit trail — every decision logged, every output traceable to its input. That traceability is not just for compliance — it's how we debug at speed. Any artefact, any decision, any output — you can trace the full lineage in seconds.

The governing principle: Truth stays in the systems of record. Reasoning is governed by AI. Actions are always human-approved. That's the Governance Lock."
